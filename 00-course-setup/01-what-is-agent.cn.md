## 什么是 AI？

[AI](https://www.oracle.com/cn/artificial-intelligence/what-is-ai/)，即人工智能，指的是接受过训练且可以模拟人类智能的计算系统。这种模拟人类智能的能力如今被广泛应用在各行各业中，例如客户服务、个性化营销、财务助手。

## 什么是 Agent？

[AI Agent](https://www.oracle.com/cn/artificial-intelligence/ai-agents/#what-are-ai-agents) 指的是一个具备 AI 能力，且能够**接受任务**、**检查环境**、**根据角色执行任务**并**根据经验调整执行方式**。

根据特定的需求，人们为 AI Agent 设定了目标。有了任务，Agent 就可以根据训练获得的知识和特定应用所需的经验来**指定计划**、**执行任务**并**实现目标**。

## Agent 的工作原理

Agent 其实更像一个规划者，它不再是一个简单的通过 prompt 生成回复的工具。Agent 需要根据分配给他的**目标**识别出实现目标所需的任务或步骤。
以一个常见的商城客服 Agent 为例。当用户想要问他指定的货物什么时候能送达时，Agent 需要规划好它需要做的事：查询用户指定订单的信息、查询货物实时位置、根据经验和环境判断货物的预计抵达时间，最后将有用的信息回复给用户。

## Agent 面临的挑战

1. 通用性。一个 Agent 往往需要明确的身份和任务，就像你很难用 Copilot 去帮你处理写代码以外的任务。
2. 复杂任务。对于简单、机械的任务而言，Agent 处理起来会得心应手，比如根据今日处理的文档来生成每日工作小结。但随着任务变得复杂，比如一些需要人类高度参与的工作中，这些工作往往不定型，这时候 Agent 就会变得难以设计和维护。对于这些任务来说，更需要把每一步都走好。
3. 依赖高质量数据。为保证 Agent 效果和可信（对于用户来说，可信往往是最关键的要素之一），提供给 Agent 的数据必须准确、及时且可用。
4. 安全风险。对于用户来说，保密也是最关键的要素之一，有些事不上称没有二两重，上了称，一千斤也打不住。商业秘密往大了说价值不可估量。因此安全团队必须时时刻刻学习新技能，以保证数据不会从任何途径泄露。

## Agent 的构成

1. 执行。让 Agent 与外部环境交互，比如帮用户变成、写邮件等等。
2. 记忆。让 Agent 记住过去的信息，通过合理地使用这些信息，Agent 可以做出更加明智的决策。
3. 推理。让 Agent 能够识别多种选择，根据可用信息做出决策。
4. 目标、效用。目标定义了期望 Agent 给出的结果，效用衡量 Agent 实现目标的程度。比如让 Agent 进行用户个性化营销，最终通过点击率来衡量 Agent 执行的效果。
5. 感知。Agent 可以通过一些机制来捕获外界信息，比如用户的文字输入、语音输入等。

## Agent 的类型

在 Agent 开发中，有这样几种常见的类型：

1. 任务导向型 Agent（Task-Oriented Agent）：这类 Agent 存在明确的任务，如编程、数据分析。
2. 规划与推理型 Agent（Planning & Reasoning Agent）：这类 Agent 强调自主分解任务、制定计划并执行。
3. 探索与学习型 Agent（Exploration & Learning Agent）：这类 Agent 强调在与环境的交互中学习，能够根据结果反思以调整执行策略。
4. 多 Agent 系统（Multi-Agent System）：这种系统由多个 Agent 组成，他们可以是上述任意的一种或多种 Agent 组成。
