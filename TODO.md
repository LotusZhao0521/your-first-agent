## 项目路线 & 执行计划（your-first-agent）

> 目的：帮助你快速了解「这个项目能学什么、怎么学」，同时也记录作者/贡献者的开发计划，方便后续维护与扩展（也方便 Cursor 恢复上下文）。

---

### 1. 项目整体目标（你能学到什么）

- **面向对象**：完全不懂代码的小白（可以看懂一点 Python 更好，但不是硬性要求）。
- **核心场景**：通用型 Agent 应用 + 统一 MCP 工具层，例如：信息整理助手、问答助手、财经资讯解读助手等。
- **对比框架**：LangChain、LangGraph、CrewAI、openai-agents-sdk 等常见 Python Agent 框架。
- **技术栈约定**：Python + `uv` 做包管理；统一在根目录管理依赖；**所有框架统一通过 MCP 使用外部工具**。
- **最终形态**：一个 GitHub 仓库 + 配套文档，读者只需复制命令即可体验从单 Agent 到多 Agent、从无工具到有 MCP 工具的完整学习路径。

---

### 2. 分阶段大计划（高层路线）

- **第 0 层：环境准备 & 基础概念（目录：`00_导言/`）**

  - 用图文 + 极简代码，让小白完成：
    - 安装 Python 与 `uv`，并能在根目录执行 `uv sync`。（对应：`00_导言/01_环境配置.md`）
    - 运行第一个 LLM 调用脚本 / Notebook，体验「模型会说话」。（示例脚本位置后续在仓库中补充）
    - 用生活化故事理解：LLM、Agent、多 Agent 协作的基本概念。（对应：`00_导言/02_什么是 Agent.md`）
    - 了解 MCP 是什么、为什么要统一用 MCP 当「工具层接口」。（对应：`00_导言/03_什么是 MCP.md`）

- **第 1 层：任务导向型单 Agent 示例（目录：`01_single_agent_examples/`，后续创建）**

  - 统一若干「小而清晰」的任务场景，例如：
    - 整理一段长文本，输出要点清单；
    - 根据指定问题，从财经新闻 MCP 中检索相关内容并生成解读。
  - 在多个框架中分别实现同一任务的「单 Agent 版本」，让读者体验不同框架的“手感”。

- **第 2 层：小型多 Agent 协作（2–3 个 Agent）（目录：`02_multi_agent_team/`，后续创建）**

  - 场景示例：检索 Agent + 分析 Agent + 汇总 Agent 的协作流程（可结合财经新闻 MCP 或其他通用信息源）。
  - 先在多 Agent 友好的框架（如 CrewAI / LangGraph）实现，再扩展到其他框架。

- **第 3 层：带 MCP 工具的复杂多 Agent 流程（目录：`03_tool_rich_pipeline/`，后续创建）**

  - 场景示例：结合多个 MCP 工具（如财经新闻、向量检索、模板渲染），完成一个「从信息采集 → 分析 → 报告生成」的流程。
  - 展示更接近真实生产环境的工作流设计，并对「什么时候该用多个 Agent、什么时候就一个 Agent + 多工具」给出直观感受。

- **第 4 层：跨框架对比与选型（目录：`04_framework_comparison/`，后续创建）**

  - 整理不同框架在：
    - 上手难度
    - 多 Agent 支持
    - 工程化能力
    - 生态 / 文档
  - 上的优缺点，对小白给出选型建议。

- **第 5 层：工程化 & 可持续维护（目录：`05_engineering_maintenance/`，后续创建）**
  - 记录日志、简单评估、部署方式（本地脚本 / 命令行 / 简单服务）。
  - 制定依赖版本更新和文档更新的流程（如每季度扫一遍主要框架的 release）。

### 4. 约定与注意事项

- **统一包管理**：所有示例通过 `uv` 管理依赖，不再使用裸 `pip`。
- **示例尽量保持「任务一致」**：同一层级（单 Agent / 小团队 / 流水线）在不同框架中尽量做同一件事情，方便对比。
- **统一工具层接口**：所有需要访问外部数据 / 能力的示例，优先通过 MCP 工具来实现，避免在各个框架里各写一套不兼容的工具。
- **对小白友好**：所有 README 和文档优先解释「怎么跑、能看到什么、可以改哪」，暂时不过度深入底层实现细节。
- **版本记录**：后续增加/修改依赖或框架版本时，更新：
  - 根目录的 `pyproject.toml`。
  - `05_engineering_maintenance/changelog_guide.md` 中的说明。
